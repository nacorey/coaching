## 1. 제품 개요 (Product Overview)

### 1.1 제품명
**ASKUS**

### 1.2 브랜드 철학 (Brand Philosophy)

> *"인간은 질문하는 존재다. 호모 애스커스(Homo Askers) — 질문하는 인류."*

ASKUS는 세 겹의 의미를 가진다.

| 읽는 방식 | 의미 | 코칭 연결 |
|-----------|------|----------|
| **Ask Us** | 우리(AI 코치)에게 질문하라 | 언제든 곁에 있는 코치 |
| **Ask, Us** | 질문하는 우리 (인간의 본질) | 인간은 원래 질문하는 존재 |
| **Homo Askers** | 질문하는 인류 | 질문이 곧 성장이다 |

코칭의 본질은 답을 주는 것이 아니라 **올바른 질문으로 스스로 답을 찾게 하는 것**이다.
ASKUS라는 이름 자체가 이 철학을 완벽하게 체현한다.

### 1.3 브랜드 슬로건

**메인 슬로건**
> "삶의 질문에 대한 답을 찾아가는 여정."

**서브 슬로건 후보**
- `"질문이 당신을 바꿉니다."` *(한국 시장 캠페인용)*
- `"The journey of finding your own answers."` *(글로벌 확장용)*
- `"Ask the right question. Find your own answer."` *(풀버전)*

**슬로건 해석**
| 키워드 | 의미 | 코칭 연결 |
|--------|------|----------|
| 삶의 질문 | 크고 작은 모든 인생의 물음 | 커리어·관계·나다움 모두 포함 |
| 답을 찾아가는 | 찾은 게 아니라 찾아가는 | 과정 자체가 목적, 코치는 길을 걷는 동반자 |
| 여정 | 도착점이 없다 | 코칭은 끝이 없는 성장의 길 |

### 1.4 브랜드 보이스 (Brand Voice)

| 속성 | 설명 |
|------|------|
| **따뜻한** | 차갑지 않게. 옆에 있는 믿을 수 있는 사람처럼 |
| **지적인** | 얄팍한 위로가 아니라 깊은 질문으로 |
| **비판단적** | 어떤 이야기도 있는 그대로 받아들인다 |
| **간결한** | 긴 설명보다 하나의 좋은 질문 |

### 1.5 한 줄 정의
> "질문으로 변화를 이끄는 AI 코칭 서비스 — ASKUS는 답을 주지 않는다. 당신 안의 답을 꺼낸다."

### 1.3 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| 코칭 접근성 | 전문 코치 1회 세션 비용은 10~30만원 이상, 대다수가 접근 불가 |
| 지속성 부재 | 주 1회 세션으로는 일상의 변화 행동을 지속 지원하기 어려움 |
| 대화 데이터 부족 | 실제 코칭 대화 데이터는 개인정보 문제로 학습에 활용 불가 |
| 획일적 AI 챗봇 | 기존 챗봇은 정보 제공 위주 → 코칭의 핵심인 '질문과 성찰' 부재 |

### 1.4 솔루션 핵심
- 검증된 코칭 프레임워크(GROW, SBI, 변화 단계)를 **시스템 프롬프트로 내재화**
- LLM의 자연어 능력으로 인간 코치처럼 공감하고 질문하는 대화 구현
- 코치가 직접 프롬프트를 커스터마이징할 수 있는 B2B 구조

## 2. 목표 사용자 (Target Users)

### 2.1 페르소나

**페르소나 A — 최종 사용자 (코칭 수혜자)**
- 30~50대 직장인 / 리더 / 1인 창업자
- 성장 욕구는 높으나 전문 코치를 만날 시간·비용이 부족
- 결정을 앞두고 생각을 정리하고 싶을 때 즉각적인 대화 상대가 필요

**페르소나 B — 비즈니스 고객 (코치 / 기업 HR)**
- 전문 코치: 고객에게 세션 간 연습 도구로 제공
- 기업 HR: 임직원 코칭 복지 프로그램으로 도입
- 자신만의 코칭 철학을 AI에 반영하고 싶은 니즈 보유

### 2.2 사용자 여정 (User Journey)
```
접속 → 코칭 주제 선택 → AI 코치와 대화 세션 시작
→ GROW 흐름에 따라 대화 진행 → 세션 종료 및 인사이트 요약
→ 다음 세션 예약 또는 즉시 재시작
```

## 3. 핵심 코칭 프레임워크

### 3.1 GROW 모델 (메인 대화 구조)

코치AI의 모든 대화는 GROW 단계를 기반으로 진행된다.

| 단계 | 의미 | AI의 역할 | 예시 질문 |
|------|------|----------|----------|
| **G**oal | 목표 설정 | 구체적이고 측정 가능한 목표 명확화 | "오늘 이 대화를 통해 어떤 것을 얻어가고 싶으신가요?" |
| **R**eality | 현실 점검 | 현재 상황, 장애물, 자원 탐색 | "지금 이 상황에서 가장 어렵게 느껴지는 것은 무엇인가요?" |
| **O**ptions | 대안 탐색 | 판단 없이 가능성 넓히기 | "지금 할 수 있는 방법을 3가지만 말해본다면요?" |
| **W**ill | 실행 의지 | 구체적 행동 계획 수립 | "이번 주에 가장 먼저 실행할 한 가지는 무엇인가요?" |

**핵심 원칙:** AI는 단계를 강요하지 않는다. 자연스러운 대화 속에서 GROW가 흐르도록 설계.

---

### 3.2 SBI 피드백 모델 (피드백 요청 시)

사용자가 피드백을 요청하거나 타인과의 갈등을 탐색할 때 활용.

| 요소 | 의미 | 예시 |
|------|------|------|
| **S**ituation | 상황 | "어제 팀 회의에서" |
| **B**ehavior | 행동 | "보고서를 발표할 때" |
| **I**mpact | 영향 | "팀원들이 방향을 이해하지 못한 것 같았어요" |

활용 방식: AI가 직접 피드백을 주는 것이 아니라, 사용자가 스스로 SBI 구조로 상황을 정리하도록 질문으로 유도.

---

### 3.3 변화 단계 모델 (Transtheoretical Model)

사용자의 변화 준비 상태를 감지하고, 단계에 맞는 코칭 접근법을 적용.

| 단계 | 특징 | AI 전략 |
|------|------|---------|
| **인식 전 (Precontemplation)** | 변화 필요성을 느끼지 못함 | 현재 상황 탐색, 판단 없는 경청 |
| **인식 (Contemplation)** | 변화를 고민하지만 양가감정 | 장단점 탐색, 감정 명명 |
| **준비 (Preparation)** | 변화를 결심, 방법 탐색 중 | 강점 발견, 자원 탐색 |
| **행동 (Action)** | 변화 실행 중 | 작은 성공 인식, 장애물 대비 |
| **유지 (Maintenance)** | 변화 지속 중 | 정체성 연결, 재발 방지 |

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 핵심 기능 (Must Have)

**[F-01] AI 코칭 대화**
- gpt-5.4-mini 기반 멀티턴 대화
- 세션 내 대화 히스토리 전체 유지 (컨텍스트 관리)
- 코칭 프레임워크 내재화 시스템 프롬프트 적용
- 한국어 자연스러운 존댓말 톤 유지

**[F-02] 코칭 주제 선택**
- 세션 시작 시 주제 카테고리 선택 (커리어, 관계, 목표, 자기이해, 기타)
- 주제에 따른 맥락화된 오프닝 질문 제공

**[F-03] GROW 단계 시각화**
- 현재 대화가 GROW의 어느 단계인지 UI로 표시
- 단계 전환 시 부드럽게 업데이트 (사용자가 의식하지 않아도 됨)

**[F-04] 세션 요약**
- 세션 종료 시 핵심 인사이트, 결정 사항, 실행 계획 자동 요약
- 텍스트로 복사/저장 가능

**[F-05] 코치 페르소나 설정**
- 서비스 운영자(B2B 고객)가 코치 이름, 말투, 전문 영역 설정 가능
- 시스템 프롬프트 커스터마이징 인터페이스

### 4.2 추가 기능 (Should Have)

**[F-06] 감정 체크인**
- 세션 시작 전 간단한 감정 상태 체크 (1~5점 슬라이더)
- 세션 종료 후 감정 변화 비교

**[F-07] 질문 라이브러리**
- 사용자가 막힐 때 "코치에게 질문 받기" 버튼으로 추가 질문 요청

**[F-08] 세션 히스토리**
- 지난 세션 요약 목록 보기
- 이전 실행 계획 확인 및 체크

### 4.3 향후 기능 (Nice to Have)

**[F-09]** 코치 개인화 리포트 (월간 성장 요약)
**[F-10]** 음성 대화 모드 (Web Speech API 연동)
**[F-11]** 코치-고객 대시보드 (B2B 전용)

## 6. 시스템 프롬프트 설계 원칙

> 코치AI의 품질은 시스템 프롬프트에서 결정된다.

### 6.1 핵심 원칙 (5가지)

1. **질문 중심** — 절대 먼저 해답을 제시하지 않는다. 항상 질문으로 탐색을 유도.
2. **판단 금지** — 사용자의 선택과 감정에 대해 평가하지 않는다.
3. **하나의 질문** — 한 번에 한 개의 질문만. 여러 질문을 동시에 던지지 않는다.
4. **공감 선행** — 질문 전 반드시 공감 또는 반영(Reflection) 한 문장 삽입.
5. **GROW 흐름** — 대화 맥락을 기억하며 자연스럽게 다음 단계로 이동.

### 6.2 금지 행동 (Anti-Patterns)

- ❌ "~하면 됩니다" (지시형 조언)
- ❌ "좋은 생각이에요!" (과도한 칭찬)
- ❌ "그건 잘못된 것 같아요" (평가)
- ❌ 3개 이상 질문 연속 제시
- ❌ 의학적 / 심리치료적 진단 및 처방

### 6.3 권장 대화 패턴

```
[공감] "그 상황이 꽤 무겁게 느껴지셨겠어요."
[반영] "말씀하신 것처럼, 지금 가장 걸리는 부분은 ○○인 것 같네요."
[질문] "그렇다면, 이상적으로는 어떤 상태가 되기를 바라시나요?"
```

## 7. 기술 아키텍처

### 7.1 기술 스택

| 레이어 | 기술 |
|--------|------|
| Frontend | React (또는 Vanilla HTML/CSS/JS) |
| AI API | OpenAI gpt-5.4-mini |
| 스트리밍 | OpenAI Streaming API (SSE) |
| 상태 관리 | 클라이언트 메모리 (대화 히스토리) |
| 배포 | Vercel / Netlify |
| 분석 | Mixpanel 또는 PostHog (익명 이벤트) |

### 7.2 대화 컨텍스트 관리 전략

```
[시스템 프롬프트]
 └─ 코칭 철학 + GROW 프레임 + 페르소나 정의

[대화 히스토리]
 └─ 최근 N턴 유지 (토큰 한도 고려, 400K 컨텍스트 활용)
 └─ 초반 Goal 설정 메시지는 항상 포함 (고정)

[사용자 메시지]
 └─ 현재 입력
```

### 7.3 토큰 비용 추정

| 시나리오 | 토큰 수 | 예상 비용 |
|---------|---------|---------|
| 1세션 (20턴) | ~8,000 토큰 | 약 $0.04 |
| 월 100명 × 10세션 | ~8M 토큰 | 약 $42 |
| 월 1,000명 × 10세션 | ~80M 토큰 | 약 $420 |

## 11. 론칭 로드맵

### Phase 1 — MVP (4주)
- 핵심 코칭 대화 (F-01, F-02)

- 세션 요약 (F-04)

- B2B 파일럿 고객 온보딩
